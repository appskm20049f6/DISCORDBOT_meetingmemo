# 內部任務看板系統（DISCORDBOT_meetingmemo）

把 Discord 各遊戲專案討論群的文字訊息，整理成「可分配、可追蹤、可通知」的工作看板。

---

## 一、這是什麼

三個部分串成一條龍：

```
Discord 各討論群文字
      │  ① 收集 bot（discord.js）
      ▼
   output/*.md  ──②  Cowork/Claude 整理成任務 ──③ 匯入──▶  內部看板網頁
                                                              │
                                            ④ 自動同步 / 手動推送
                                                              ▼
                                   Discord 各專案「#工作提醒」頻道（看板 + 通知）
```

| 元件 | 角色 |
|------|------|
| **收集 bot** | 定時撈各討論群新訊息，存成 Markdown |
| **看板網頁** | 人 → 專案 → 狀態三層；分配、改狀態、回報、推送 |
| **Discord #工作提醒** | 每個專案一個頻道，自動顯示該專案任務看板；推送通知也發這裡 |
| **Cowork** | 每天讀新訊息、整理成任務、匯入、通知（見 [COWORK.md](COWORK.md)） |

- 看板網址（內網）：**http://10.1.178.130:3100**

### 專案涵蓋範圍（兩個層級）
| 層級 | 涵蓋專案 | 說明 |
|------|---------|------|
| **全部專案皆有「工作提醒」板＋每日自我提醒** | 全部 9 個遊戲專案 | 任何人在該專案建任務、指派給自己，每天 10:00/17:00 就會被自動提醒（推到該專案 `#工作提醒` 並 tag） |
| **自動收集對話＋整理成任務＋自動派工** | 僅 4 個正式專案：**神州M、霸王之業、幻世錄重製版、天使之戀國際版** | 只有這 4 個會去搜討論群、用 AI 整理新工作、自動分配負責人 |

- 要把某個專案從「只有提醒」升級成「自動整理」，需把它的討論群加進 `config.json` 的 `channels`、並納入排程指令（跟維護者說即可）。
- 新增任務時的「專案分類」是**下拉選單，只能從現有專案選**（不開放自訂新專案，維持受控）。

---

## 二、看板網頁怎麼用

打開 **http://10.1.178.130:3100**，左中右三區、右側任務再分四欄：

```
┌── 成員 ──┬── 專案分類 ──┬──── 任務（依狀態四欄）─────────────────┐
│ 👤 成員A [新]│ 📁 專案一[新]│ 尚未開始 │ 進行中 │ 已完成 │ 已封存  │
│ 👤 成員B   │ 📁 專案二   │ [卡片]   │ [卡片] │ [卡片] │ [卡片]  │
└──────────┴─────────────┴────────────────────────────────────────┘
```

### 基本操作
1. **點左側某人** → 中間出現「他的專案分類」
2. **點某個專案** → 右側顯示該專案任務，分「尚未開始／進行中／已完成／已封存」四欄
3. **紅色「N 新」徽章** = 有尚未看過的新任務；點進該分類就會消失
4. **拖曳卡片** 到另一欄 = 直接改狀態（會即時存檔）
5. **已封存**：標記「已完成」滿 3 天會自動移到此欄，保留可反查、不再提醒、不洗版

### 任務卡片內容
- 標題、📅 日期、說明、是否已回報
- 逾期未完成的任務：日期會變**紅字（已逾期）**

### 點開任務（編輯視窗）
- 改 **標題 / 說明 / 負責人 / 專案分類 / 狀態 / 日期**
- 右側 **月曆**：藍色=今天、紅色=工作日期；可直接點月曆某天設定日期
- **完成回報**：在說明框打字，或游標放框內 **Ctrl+V 貼上截圖**，按「標記完成並回報」
- **🔔 推送通知**：發到該專案 `#工作提醒` 並 tag 負責人（送出前會跳確認）
- **刪除**：刪掉這筆任務

### 工具列按鈕
| 按鈕 | 功能 |
|------|------|
| ＋ 新增任務 | 手動建立一筆 |
| ⤵ 匯入 JSON | 貼上整理好的任務陣列批次匯入 |
| ＋ 新增成員 | 手動加人（建議用 `npm run members` 從 DC 同步） |
| 🔔 整個專案一次推送 | 把某人某專案的未完成任務整理成一則發出 |

---

## 三、推送通知說明

- 按鈴或「整個專案一次推送」→ 發到該專案的 **`#工作提醒`** 頻道，並 **@tag 負責人**
- **同一個人重複推送會自動取代**：先刪掉上一則給他的舊通知，只留最新一則，不洗版
- 只有從 DC 同步來的成員才能被 tag（手動新增的成員只會顯示名字）

---

## 四、每天的流程（交給 Cowork）

每天 **10:00 / 17:00**，請 Cowork 讀 [COWORK.md](COWORK.md) 執行：收集新訊息 → 整理成任務（含負責人/狀態/日期）→ 匯入看板 → 通知執行人。

要手動跑收集也可以：`npm run collect`（增量，只撈上次之後的新訊息）。

---

## 五、常用指令

```powershell
# 收集
npm run collect          # 撈各頻道新訊息（增量）
node src/bot.js --recent 30   # 撈近 30 天（指定頻道：再接頻道ID）

# 看板與成員
npm run web              # 啟動看板網頁（PM2 已常駐，平常不用手動開）
npm run members          # 從 DC 同步成員到看板
npm run channels         # 列出頻道 ID
npm run groups           # 列出分類（工作群組）結構

# Discord 工作提醒看板
npm run setup-boards     # 建立工作提醒頻道（全部群組；指定用 node src/bot.js --setup-boards <分類ID...>）
npm run update-boards    # 手動更新一次看板訊息
```

### 服務維運（PM2）
本系統用 PM2 常駐，app 名稱 `memo-web`、`memo-bot`（與其他專案 `metagooglead-*` 互不影響）：
```powershell
npx pm2 status                      # 看狀態
npx pm2 restart memo-web memo-bot   # 改完程式後重啟套用
npx pm2 logs memo-bot               # 看 bot 日誌
```
- **常駐 bot** 會：每天 09:00/18:00 自動收集、看板每 10 分鐘自動更新、bot 顯示上線
- **開機自動還原**：已設定（啟動資料夾的 `pm2-resurrect-boot.vbs`），重開機會自動把服務救回
- 新增/移除 PM2 程序後要 `npx pm2 save` 才會記住

## 六、資料與設定位置

| 路徑 | 內容 |
|------|------|
| `config.json` | 要撈的頻道、排程時間、專案→討論群、工作提醒看板群組 |
| `.env` | DISCORD_TOKEN、PORT、BOARD_URL |
| `data/db.json` | 看板的成員與任務（核心資料） |
| `data/boards.json` | 各專案工作提醒看板的頻道/訊息位置 |
| `data/notifylog.json` | 各人最後一則通知（用來取代舊通知） |
| `data/uploads/` | 完成回報貼上的圖片 |
| `output/<日期>/` | 收集到的討論群 Markdown |
| `COWORK.md` | 給 Cowork 的每日任務說明書 |

---

## 七、設定新專案的看板（之後要擴充時）

1. `npm run groups` 找該遊戲分類的 categoryId
2. `node src/bot.js --setup-boards <categoryId>` 建該群的 `#工作提醒`
3. （選用）在 `config.json` 的 `projectChannels` 補「專案名→討論群ID」
4. 在 `config.json` 的 `channels` 補該討論群，之後 `npm run collect` 就會一起撈
